Problem
에이전트는 왜 잊을까?
에이전트 하네스에서 기억 계층만 아직 표준이 없습니다.
1

세션 간 기억 부재

매 세션마다 아키텍처, 패턴, 이력을 처음부터 다시 설명. 기업 규모에서 연간 수만 달러의 토큰 낭비.

2

에이전트 간 기억 단절

Claude Code의 지식을 Gemini CLI에서, 에이전트 A의 학습을 에이전트 B에서 활용 불가. 기억 사일로.

3

하네스 기억 계층 부재

기존 솔루션은 명시적 검색만, 특정 런타임 종속, LTM 단일 계층. 표준화된 기억 인프라가 없다.

Solution
memtomem이 해결합니다
STM + LTM 분리 아키텍처로 인간 유사 기억 체계를 에이전트에 구현.
S

능동적 서피싱

에이전트 요청 없이 관련 기억을 자동 주입. 5단계 관련성 게이팅 + 피드백 자동 튜닝.

G

MCP 프록시 게이트웨이

STM이 에이전트-서버 사이 프록시로 동작. 코드 수정 없이 모든 MCP 통신에 기억 자동 주입.

C

크로스 런타임 동기화

Context Gateway가 에이전트 정의·스킬·커맨드를 6종 런타임에 자동 동기화.

10

10종 압축 전략

콘텐츠 유형별 자동 선택. progressive delivery로 제로 정보손실 옵션 제공.

A

멀티 에이전트 지식 공유

네임스페이스 격리 + 공유. Agent→Agent, Human→Agent 전방향 지식 흐름.

L

완전 로컬 구동

SQLite + ONNX. GPU 불필요, 외부 서비스 불필요. 데이터 주권 100% 확보.
